二分搜索

Jiangnan CAI
2022-08-04

本篇文章的二分搜索并不局限于数组上的二分搜索，还包括树数据类型的二叉搜索树，拓展到k维数据的kd-tree，其回溯的优化BBF，乃至适用于更高维度的ball-tree之类的的，应用二分思想的算法。本文将做一个综合的介绍。

我会将二分搜索的算法的关键归纳为2点：

  • 对区域的划分
    区域的划分依赖于数据的有序性。

    • 在一维数组里，数据要升序或者降序排列；
    • 二叉搜索树的构造定义就决定了：左节点< 根节点 < 右节点；
    • 在kd-tree里，找到了方差最大的维度，也是要对该维度的所有数据进行排序，选择一个中位数来对数据进行二分；
    • 突然发现一个问题，ball-tree好像在区域划分的时候好像并不依赖于数据的有序性，是根据距离来作为区域划分依据的。

  • 对数据的距离求解：

    • 在一维数据中，最简单了，就是相当于两个数比较大小，差值越小，越接近
    • 在多维数据中，有多种距离度量

可不可以这么认为，有序性，算是数据集在一维上的有效特征，但是在多维的话，就没有效果了，ball-tree直接采用了高维的特征来构建二叉树，

一维元素二分搜索

有序 + 顺序存储

binary search （array）

这个常见的搜索方法，我们非常熟悉，应用于数组中，就是通过不断的与Mid值进行比较，缩小区间范围，来降低搜索时间的方法。不过这种方法对于数组内部的数据是要求**有序的**。其时间的复杂度为 $O(\log n)$。

有一些针对二分搜索的优化，第一种就是不一定要二分，如果数组是均匀的，可以根据要搜索的数在整个数组的大概范围来决定首次搜索的值，类似插值搜索。还有一种，从实现方式上的，用递归来实现，不过只会走一条线路，我觉得和普通写法没有什么区别。

binary search tree

就如同前面说的那样，二叉搜索树有一个重要的特性：左节点 < 根节点 < 右节点，这也是二叉搜索树的数据有序性，通过中序遍历，我们可以构造一个有序的数组。某种意义上，二叉搜索树的相当于将数组的二分搜索进行了可视化，

一般来说，对于二叉树这种数据结构，我们都会遇到两个问题：

  • 树的构造（array -> tree）
  • 树的增删改查

二叉搜索树也不例外，上面这两点都是算法中考察的重点。而且因为二叉搜索树的有序性，增删改就变得非常的麻烦，因为涉及到树结构的变化。

这周得重新复习一下二叉搜索树的各种操作了。

多维数据二分搜索

kd-tree和ball-tree都在python的sklearn库里有实现。

kd-tree

在这次的行为分类中，本来一开始是想用kd-tree的，但是数据的维度太多了，最后还是用了ball-tree，因为看大家都推荐20维度以下才使用kd-tree，20维以上都推进啊使用ball-tree。这个算法也经常用于激光点云之类的多维数据中。

方差展示了数据的离散度，为什么一开始选择方差大的维度进行划分？
方便后期回溯？

这次选择了这个维度，下次还可以选择同一个维度吗？就是还需要重新计算当前维度的方差吗？（也就是经过一次二分的剩余数据的方差），那这个kd-tree的层数，不一定等于数据的维度，因为有时候是同一维度进行多次二分，有时候是不同维度进行二分。
一个维度其实不一定只会被二分一次，如果方差过大，有可能会被多次二分。

原理

  • 构造： 二分，计算每个维度的方差，然后选择方差最大的维度，对维度内数据排序然后选择中位数作为划分的标准。这个中位数实际上相当于一个超平面，将数据集一分为二。
  • 搜索： 近邻，最近邻或者k近邻，根据任务决定，找匹配点可能用最近邻，要是分类的话可能会用k近邻

构造树的时候用到了二分法，搜索的时候用到了近邻法。
